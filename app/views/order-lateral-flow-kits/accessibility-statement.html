Accessibility statement for the Order COVID-19 rapid lateral flow tests service

The purpose of the service is to allow a user to order a COVID-19 rapid lateral flow test.

This web platform is run by NHS England. We're committed to making the website as accessible as possible for people with visual, hearing, cognitive and motor impairments.

This means you should be able to:

  • change colours, contrast levels and fonts
  • zoom in up to 400% without the text spilling off the screen
  • navigate most of the website using just a keyboard
  • navigate most of the website using speech recognition software
  • navigate the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

We've also made the website text as simple as possible to understand.
My Computer My Way has advice on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of the websites are not fully accessible:

  • occasionally the reCAPTCHA security check may be inaccessible to some users
  • parts of the service cannot be completed when JavaScript is disabled while using Internet Explorer

Feedback and contact information

If you have feedback or a complaint about the accessibility of this service, contact us online at https://enquiries.test-and-trace.nhs.uk/.

You can also call 119. Check 119 opening times.

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’). If you're not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about the accessibility of these websites

NHS England is committed to making these websites accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

These websites are partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard due to the non-compliances listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

reCAPTCHA security check on the version of reCAPTCHA our service occasionally uses has an audio alternative, but this is not accessible to users. This fails WCAG 2.1 success criteria 2.1.1 (A), 2.1.3 (AAA), 2.5.6 (AAA), 4.1.1 (A)

There are parts of the service that cannot be completed when JavaScript is disabled while using Internet Explorer. This fails WCAG 2.1 success criterion 4.1.1 (A). The service was designed to operate in this way and there are no plans in place to update this

What we're doing to improve accessibility

We try to make our services as accessible as possible. However, we might have missed something, so please let us know if you find any accessibility issues using the contact details in the ‘Feedback and contact information’ section.

In early 2023 we commissioned a Web Content Accessibility Guidelines (WCAG) version 2.1 accessibility audit of this website. The audit was carried out by Dig Inclusion who are accessibility specialists. They looked at a selection of pages most likely to have accessibility concerns. The audit found some accessibility issues, which we have now fixed.

We have also established accessibility processes, which we use as part of our ongoing work. These processes help us to design and develop accessible content when we make a change to any of our websites.

Preparation of this accessibility statement

This statement was prepared on 27 October 2020. It was last reviewed on 25 September 2023.

This website was last tested in August 2023. The test was carried out by Dig Inclusion.
